## 各系统核心区别是什么？

说到底，搬瓦工系统选择跟 VPS 速度没关系。速度只跟线路有关——CN2 GIA 比 CN2 GT 快，跟你装 Ubuntu 还是 Debian 没半点联系。系统影响的是**用起来的感受**：教程好不好查、软件包新不新、系统稳不稳。

### Debian

稳定。这两个字基本概括了 Debian 的特点。

Debian 的软件版本不是最新的，但特别可靠。装好之后可以跑好几年不出问题，安全更新按时推送，基本上不需要操心。资源占用也很低，对于 512MB 内存的 VPS 来说友好。

两个明显短板：软件包版本比 Ubuntu 旧一些，某些依赖较新版本的脚本或者软件可能装不上。另外搜教程的时候，Debian 的中文资料比 Ubuntu 少一截，遇到问题自己查起来会稍微麻烦一点。

目前搬瓦工 KiwiVM 后台推荐的是 Debian 13（代号 Trixie），已经是最新 stable，新部署优先看这个版本。

### Ubuntu

教程最多。这是 Ubuntu 在服务器场景里最大的竞争优势。

碰到问题搜一下，基本都能找到 Ubuntu 版本的解决方案，对新手来说这点特别实在。软件包版本也比 Debian 更新，很多依赖 Python 新版本或者特定 Node.js 的项目，在 Ubuntu 上装起来更顺畅。

搬瓦工刚在 KiwiVM 后台加上了 Ubuntu 26.04 LTS，是 2026 年 4 月底 Canonical 正式发布之后没多久就同步进来的。内核升到了 Linux 7.0，对于想用最新环境的人来说这个是加分项。不过如果你现有的 24.04 跑得很稳，没必要急着换。

有一个小地方要注意：Ubuntu 的 Snap 包管理机制，有时候会让部分软件的首次启动慢一些。算是小问题，不影响主要用途。

### Rocky Linux / AlmaLinux

这两个是 CentOS 停掉之后社区推出的替代品，100% 兼容 RHEL 生态。

如果你之前用过 CentOS，想平滑迁过来，选 Rocky Linux 成本最低，配置基本不用改。生命周期长，支持周期可以到 2032 年左右，适合不想频繁升级系统的生产环境。

反过来说，如果你是新手，没有 CentOS 历史包袱，直接选 Rocky Linux 反而增加了学习成本——包管理用 `dnf`，防火墙用 `firewalld`，跟 Debian/Ubuntu 的 `apt` + `ufw` 体系不同，网上教程对上号会稍微麻烦点。

## 搬瓦工系统选择：按场景直接给答案

问我"搬瓦工装哪个系统好"，我的回答是：**大部分人选 Debian 12/13 或 Ubuntu 24.04 不会错**。具体怎么选，看用途：

**学 Linux / 折腾玩 / 新手入门**
→ Ubuntu 24.04。教程最多，出了问题最容易找到解答。能走的弯路基本都有人走过了，直接 Google 一下就有答案。

**个人建站 / 博客 / 轻量应用**
→ Debian 13。稳定、省资源，装好基本不用管它。低内存套餐（512MB / 1GB）跑起来比 Ubuntu 省。

**部署 AI 应用 / Dify / n8n / 跑 Docker**
→ Ubuntu 24.04 或 Ubuntu 26.04。Docker 官方镜像、各种 AI 工具的安装脚本对 Ubuntu 支持最好，遇到坑有更多地方可以问。

**原来用 CentOS / 企业级项目 / 有 RHEL 兼容需求**
→ Rocky Linux 9。迁移成本最低，生态熟悉，长期支持有保障。

**不知道选什么**
→ Debian 13，闭眼装。实在出问题了，重装系统在搬瓦工 KiwiVM 里是分钟级的操作，数据清完重来不是大事。

## KiwiVM 重装系统完整步骤

搬瓦工的重装系统功能在 KiwiVM 控制面板里，操作很简单，但有几个细节容易踩坑。

**重装前必做：备份数据。**重装会清空 VPS 上所有的东西，包括文件、数据库、配置，且无法恢复。如果有需要保留的内容，先用 SFTP 工具（WinSCP、FileZilla 都行）把文件拉到本地，数据库用 mysqldump 导出一份 SQL，或者用 KiwiVM 里的 Snapshot 功能打一个快照。

步骤如下：

1. 登录搬瓦工账号，进入 My Services，点开对应 VPS 的 KiwiVM Control Panel
2. 在 KiwiVM 主页（Main controls）找到 VPS 运行状态，点 **Stop**，等待关机（显示 Stopped 才算完）
3. 左侧菜单点 **Install new OS**
4. 在下拉列表里选择目标系统，比如 Debian 13 或 Ubuntu 24.04
5. 勾选下方的 **"I agree that all existing data on my VPS will be lost."**
6. 点击 **Reload**，开始重装
7. 重装完成后，页面会显示新的 **root 密码**和 **SSH 端口号**——这个页面只出现一次，务必保存好

整个过程通常 1-2 分钟内完成。如果点了 Reload 报错 "Unable to reload a running VPS"，说明 VPS 没有完全关机，回到主页再 Stop 一次再试。

## 关于系统选好之后的几件事

系统装好之后，有几件事值得做：

装完第一件事是更新软件包。Debian/Ubuntu 用 `apt update && apt upgrade -y`，Rocky Linux 用 `dnf update -y`。系统刚装完往往有一批待更新，先跑一遍再开始部署应用。

SSH 端口默认每次重装会变。重装完页面上显示的端口号要保存下来，用 Xshell 或 Terminal 连接的时候记得更新。

搬瓦工本身支持 Snapshots 功能，系统配置好之后打个快照，后续哪天搞坏了可以直接回滚，不用重新配一遍环境。

## FAQ

**Q：搬瓦工系统选择和 VPS 速度有关系吗？**

没有直接关系。速度取决于套餐线路（CN2 GIA > CN2 GT > 普通线路），跟你选 Ubuntu 还是 Debian 没关系。系统影响的是用起来的便利性、软件兼容性、以及遇到问题时能找到多少参考资料。

**Q：不知道选什么系统，能给一个最终答案吗？**

可以。如果你是新手，没有特别需求，选 **Debian 13**。资源占用低，稳定性强，支持时间够长。半年后熟悉了有明确偏好，随时在 KiwiVM 里重装，操作成本极低。

**Q：重装系统会影响 IP 地址吗？**

不会。重装系统只是换了操作系统，IP 地址不变，套餐的机房位置也不变。如果你需要换 IP，要通过迁移机房的方式来实现，跟重装系统是两件事。

**Q：Ubuntu 26.04 刚出来稳不稳，要不要装？**

搬瓦工 2026 年 4 月底刚加进来的。作为 LTS 版本，Canonical 官方承诺五年支持，内核和软件包都是最新一代。如果你的项目需要新软件栈可以直接用，如果当前 24.04 跑得好就没必要折腾——24.04 官方支持到 2029 年，完全够用。
